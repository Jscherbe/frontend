1. Basic Document List

This example demonstrates the core functionality with varied content lengths to show how the defensive min-width: 0 prevents grid blowouts. The parent defines a grid with 4 tracks (auto 1fr auto auto). The middle column spans 2 tracks (--span-2) and acts as a Subgrid automatically. On mobile, the title and metadata will gracefully stack vertically.

2. Directory with Visual Header

This example adds an optional header row. The header is purely visual (hidden from screen readers) to help alignment on desktop. On mobile, we use the mobile-columns variable to keep the layout persistent: the Avatar and Action buttons remain on the sides, while the Name and Role stack vertically in the middle.

3. Mobile Overrides: Inline Wrapping Data

By default, nested columns stack vertically on mobile to save horizontal space. However, for small data points (like tags or statuses), a tall vertical stack can look disjointed. Using the --wrap-small modifier allows these nested items to flow inline side-by-side on small screens, while still snapping to the strict column grid on desktop.
