1. Basic Document List

This example demonstrates the core functionality. The parent defines a grid with 4 tracks (auto 1fr auto auto). The middle column spans 2 tracks (--span-2) and acts as a Subgrid automatically because it contains other .data-list__column children. On mobile, the title and metadata will stack vertically.

2. Directory with Visual Header

This example adds an optional header row. The header is purely visual (hidden from screen readers) to help alignment on desktop. On mobile, the header disappears (display: none) and the columns stack. We use --hide-small on non-critical columns ("Role", "Last Active") to keep the mobile view clean.

3. Mobile Overrides (Wrap instead of Stack)

By default, nested columns stack vertically on mobile. Using --wrap-small overrides this, allowing nested items (like metadata tags) to flow naturally inline on small screens instead of creating a tall vertical stack.

4. Interactive Rows (Proxy Click)

This example demonstrates the "Clickable Row" pattern. We use the .data-list--clickable modifier for hover styles and the data-ulu-proxy-click attribute to initialize the JavaScript behavior. The click on the row is proxied to the element with data-ulu-proxy-click-source (the Title link).

5. Simple Link List (Anchor Rows)

For simple navigation lists where the entire row is a link, you can use <a> tags as the list items. The container uses a <div> instead of a <ul>, but the grid layout works exactly the same.
